# Deployment Directory Configuration

This file controls which directories are copied to the `main/` directory during GitHub Pages deployment.

## Format

Each line specifies a directory to copy:

```
source_dir:target_dir
```

- `source_dir`: The directory in the repository root to copy
- `target_dir`: The name to use in the `main/` directory (optional)

If `:target_dir` is omitted, the source directory name is used.

## Features

- **Comments**: Lines starting with `#` are ignored
- **Empty lines**: Blank lines are ignored
- **Renaming**: Use `source:target` format to rename directories during copy
- **Validation**: The script warns if a source directory doesn't exist

## Examples

```bash
# Copy microsite as-is
microsite

# Copy prvni_web and rename to prvni-web
prvni_web:prvni-web

# Copy file_giver
file_giver
```

## Usage

The `.github/scripts/copy-dirs.sh` script reads `.deploy-dirs` during deployment and automatically copies all listed directories.

To add a new directory to the deployment:
1. Add the directory name to this file
2. Commit and push
3. The workflow will automatically include it in the next deployment

To remove a directory from deployment:
1. Comment out or remove the line from this file
2. Commit and push
